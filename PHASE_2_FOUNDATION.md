# PHASE 2: FOUNDATION BUILDING

**Purpose:** Build the core elements — protagonist character and mechanism design.

**Prerequisites:** Phase 1 completed and concept validated.

---

## PROTAGONIST RULES

| DO ✅ | DON'T ❌ |
|-------|----------|
| Competent in their job | Passive victim of events |
| Has personality (quirks, opinions, self-doubt) | Blank slate narrator |
| Makes ACTIVE choices with consequences | Just reacts to things happening |
| Has internal debate before decisions | No inner conflict |
| Observes details, analyzes situations | Misses obvious things |
| Lives alone, no family mentioned | Has spouse/children/parents |
| Has PROFESSIONAL PRIDE in their work | Treats job as meaningless |

---

## CHARACTER DEPTH FORMULA

**Character depth = specific time reference + professional knowledge + emotion**

```
❌ "मैं होटल में काम करता हूं।"
✅ "मैं पिछले बारह सालों से ग्रैंड होटल में नाइट मैनेजर हूं। लोग कहते हैं — मैं आवाज से पहचान लेता हूं कौन आ रहा है। मुझे इस काम पर गर्व है। मैंने अपनी जिंदगी की सबसे अच्छी किताबें यहीं फ्रंट डेस्क पर बैठकर पढ़ी हैं।"
```

## CHARACTER BACKSTORY TEMPLATE

**Must cover ALL points in first 1,000 words of the story:**

```
1. How long at this job? (specific years/months)

2. What does protagonist KNOW that no one else knows? 
   (building quirks, patterns, secrets)

3. What is their PROFESSIONAL PRIDE? 
   (what are they best at?)

4. What is their current struggle/frustration? 
   (career decline, boredom, physical limitation)

5. What is their relationship with the setting? 
   (specific sounds, smells, routines they've memorized)

6. Show them DOING their job competently before the mystery starts
```

## THE MECHANISM (CRITICAL)

The mechanism = HOW the mystery delivers its content to the protagonist.

### Good vs Bad Mechanisms

| Good Mechanisms | Bad Mechanisms |
|----------------|----------------|
| Letters that arrive with story content | Ghost appears and talks (too vague) |
| Object that CHANGES when you interact | Phone calls from unknown number |
| Recording that plays at specific time | Dreams/visions (unreliable) |
| System log / computer that shows data | "I just knew" (no mechanism) |
| Physical object with layers (paint, paper) | Voices in protagonist's head only |

---

## MECHANISM DESIGN RULES

**6 Non-Negotiable Rules:**

1. **Mechanism must be PHYSICAL** — anyone could theoretically see/verify it

2. **Mechanism must have RULES** — specific time, trigger, conditions that can be tested

3. **Show it 5-6 times with variation** — each time reveals something DIFFERENT

4. **Each repetition must ESCALATE** — the content gets darker, the stakes rise

5. **Protagonist must TEST the mechanism at least twice** before believing

6. **Mechanism must have a LINK to the twist** — the mechanism IS part of the truth

---

## MECHANISM VARIATION TABLE

Plan 5-6 showings with this structure:

| Showing # | What Happens | Content Escalates | External Consequence |
|-----------|-------------|-------------------|---------------------|
| 1 | Discovery | Mild, ambiguous | None (protagonist alone) |
| 2 | Confirmation | Personal relevance | Protagonist loses sleep |
| 3 | Deepening | Reveals hidden information | Protagonist tells a coworker |
| 4 | Threat | Warns about danger | Authority figure dismisses it |
| 5 | Crisis | Forces a choice | Witness/coworker sees something |

**Each repetition must be DIFFERENT from the last** — the delivery method changes, the content gets darker, the stakes rise.

---

## MECHANISM EXAMPLES

```
✅ EXAMPLE 1 (Letters):
1. First letter — story sample arrives (curiosity)
2. Second letter — next pages, better story (excitement)
3. Third letter — police arrive, story matches real crimes (fear)
4. Fourth letter — protagonist changes story, letter stops (consequence)
5. Fifth letter — more pages, reveals killer's identity (crisis)

✅ EXAMPLE 2 (Painting):
1. First wash — painting shows island scene (curiosity)
2. Second wash — painting shows protagonist with arm wound (confirmation)
3. Third wash — painting shows treasure chest (greed)
4. Fourth wash — painting shows shadow figure with red eyes (threat)
5. Fifth wash — painting shows protagonist in danger (crisis)
```

## TESTING SCENES

**Testing scenes must show concrete action:**

```
✅ "मैंने अपनी कलाई घड़ी देखी। रात के 2:15 बज रहे थे। अभी दो मिनट थे। मैंने डायरी रख दी और फोन के पास खड़ा हो गया। 2:17 बजे फोन बजा।"
```

Protagonist must:
- Set up a specific test condition
- Wait and observe
- React to the result
- Try to find rational explanations
- Test again with variation

## SUPPORTING CHARACTERS TEMPLATE

### The Three Required Roles:

```
1. THE WITNESS (younger/junior)
   - Works with protagonist
   - Sees/senses something is wrong
   - Provides emotional reaction (gets scared)
   - Example: "राघव, 28 साल का जूनियर इंजीनियर। पहले हंसता है 'सर आपको भ्रम है।'
     फिर खुद देख लेता है तो डर जाता है। 'सर ये सही नहीं है। मुझे यहां से निकालिए।'"

2. THE AUTHORITY (older/superior)
   - Has power over protagonist
   - Dismisses the mystery (increases protagonist's isolation)
   - Adds external pressure
   - Example: "देशमुख साहब, 58 साल के सीनियर मैनेजर। 'शंकर, तुम बूढ़े हो रहे हो।
     डॉक्टर को दिखाओ।' वो नहीं मानता। प्रोटैगनिस्ट को अकेला कर देता है।"

3. THE KEEPER (old-timer/insider)
   - Knows the building's/situation's history
   - Holds one crucial piece of information
   - Doesn't reveal everything at once
   - Example: "सविता, 55 साल की सफाई वाली। 20 साल से यहां है।
     वो जानती है कि बीस साल पहले यहां क्या हुआ था। लेकिन बोलती नहीं।
     सिर्फ इशारा करती है। 'भैया, उस कमरे में मत जाइए।'"
```

### Character Template (for EACH supporting character):

| Field | Required | Example |
|-------|----------|---------|
| NAME | ✅ | मोहन, इंस्पेक्टर शर्मा, सविता |
| AGE/ROLE | ✅ | "दिन वाला गार्ड, 45 साल का" |
| PERSONALITY | ✅ | "बातूनी है। हर चीज में टांग अड़ाता है।" |
| ATTITUDE TO PROTAGONIST | ✅ | "शंकर जी को छोटा भाई मानता है। चिंता करता है।" |
| REACTION TO MYSTERY | ✅ | "पहले हंसता है। फिर डर जाता है। फिर दूर भागता है।" |
| PLOT FUNCTION | ✅ | "बाहरी दबाव बनाता है। प्रोटैगनिस्ट को रिपोर्ट करने को कहता है।" |
