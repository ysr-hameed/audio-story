# MASTER WORKFLOW: 5-Phase Story Generation System

**Welcome to the systematic approach to writing psychological horror stories in Hindi for TTS.**

This workflow breaks down the complex story generation process into 5 manageable phases. Each phase builds on the previous one, ensuring quality and consistency.

---

## WHY THIS WORKFLOW EXISTS

### The Problem:

Without structure, AI (and humans) tend to:
- Skip critical elements (backstory, atmosphere, supporting characters)
- Write hooks too late (boring the listener for 5+ minutes)
- Forget addiction loop phases (especially Excitement and Greed)
- Create logical inconsistencies (mechanism contradicts itself)
- Produce flat atmosphere (description without dread)
- Miss foreshadowing (twist feels like a cheat)
- Generate short stories (<5,000 words when 9,000 is needed)

### The Solution:

**Read instructions FIRST, in order. Then write with confidence.**

Each phase has:
- ✅ **Clear input requirements** (what you need before starting)
- ✅ **Specific rules and templates** (what to create)
- ✅ **Output checklist** (verify before moving forward)
- ✅ **No ambiguity** (every decision is guided)

---

## THE 5 PHASES

### **PHASE 1: CONCEPT VALIDATION** (~15 minutes)
**File:** `PHASE_1_CONCEPT_VALIDATION.md`

**Purpose:** Filter bad concepts before investing time.

**You will:**
- Check against instant rejection rules
- Validate genre (psychological horror, NOT family drama)
- Verify mechanism is physical and testable
- Confirm hook can appear in first 300 words
- Fill out concept template

**Output:** Validated concept + filled template

**Stop here if:** Concept fails any rejection rule. Choose a different concept.

---

### **PHASE 2: FOUNDATION BUILDING** (~30 minutes)
**File:** `PHASE_2_FOUNDATION.md`

**Purpose:** Build protagonist character and mechanism design.

**You will:**
- Create protagonist backstory (covering all 6 template points)
- Define protagonist's professional pride and struggles
- Design mechanism with 5-6 variations planned
- Create 3 supporting characters with full templates
- Plan testing scenes

**Output:** Complete character profiles + mechanism variation table

**Stop here if:** Backstory doesn't cover all 6 points, OR mechanism has <5 variations planned, OR any supporting character has <2 scenes planned.

---

### **PHASE 3: ARCHITECTURE** (~45 minutes)
**File:** `PHASE_3_ARCHITECTURE.md`

**Purpose:** Plan complete narrative structure.

**You will:**
- Map three-act structure with word counts
- Plan all 8 addiction loop phases (with exact Hindi dialogue)
- Design moral dilemma with tangible costs
- Choose twist type and plant foreshadowing
- Verify internal consistency (supernatural OR orchestrated, not both)
- Map real-world consequences to events

**Output:** Complete story architecture + scene-by-scene outline

**Stop here if:** Any addiction phase is missing, OR twist is on banned list, OR mechanism logic is inconsistent, OR no foreshadowing planned.

---

### **PHASE 4: WRITING EXECUTION** (~3-4 hours)
**File:** `PHASE_4_WRITING.md`

**Purpose:** Write the actual story with quality standards.

**You will:**
- Write Act 1 (2,000 words) with hook in first 300
- Write Act 2 (4,000 words) with mechanism variations
- Write Act 3 (2,000 words) with twist and confrontation
- Insert atmosphere every 300-500 words (with DREAD)
- Apply Hindi language rules and TTS-friendly substitutions
- Use detailed narration (no word-by-word style)
- Convert all numbers and English words to Hindi

**Output:** Complete story (7,000-10,000 words)

**Stop here if:** Story is <7,000 words, OR any addiction phase is missing from text, OR atmosphere is flat (description without dread), OR hook appears after 300 words.

---

### **PHASE 5: QUALITY ASSURANCE & PUBLISHING** (~1-2 hours)
**File:** `PHASE_5_QUALITY.md`

**Purpose:** Verify quality and prepare for publishing.

**You will:**
- Run 24-point verification checklist
- Check for common failure patterns
- Search and fix: digits, English words, chandrabindu
- Create SEO.md (title, description, tags, thumbnail prompt)
- Create thumbnail (1280×720 PNG)
- Generate audio with Piper TTS
- Generate video (thumbnail + audio)

**Output:** Publication-ready story + SEO + thumbnail + audio + video

**Stop here if:** Any of the 24 verification points fail. Fix before generating audio/video.

## HOW TO USE THIS WORKFLOW

### For AI Language Models:

**Step 1:** Read PHASE_1_CONCEPT_VALIDATION.md completely
**Step 2:** Generate concept and fill template
**Step 3:** Verify against rejection rules
**Step 4:** If passed → Read PHASE_2_FOUNDATION.md
**Step 5:** Continue through phases sequentially
**Step 6:** Do NOT skip ahead. Each phase builds on the previous.

### For Human Writers:

**Use this as a checklist and guide:**
- Read one phase at a time
- Complete all tasks in that phase
- Verify output checklist before moving forward
- Refer back to phase documents when writing

### For Multi-Session Work:

**You can pause between phases:**
- Complete Phase 1 → Save concept template → Resume later
- Complete Phase 2 → Save character profiles → Resume later
- Complete Phase 3 → Save architecture outline → Resume later
- But DO NOT pause mid-phase. Finish the phase first.

## COMMON MISTAKES TO AVOID

| Mistake | Why It Happens | How to Prevent |
|---------|---------------|----------------|
| Hook appears after 1,000 words | Started writing without reading Phase 1 | Read Phase 1 first, verify hook timing |
| Backstory is only 300 words | Didn't check Phase 2 template | Cover all 6 backstory points, aim for 1,000+ words |
| Excitement/Greed phases missing | Skipped addiction loop planning in Phase 3 | Map all 8 phases with Hindi text in Phase 3 |
| Atmosphere is flat | Wrote description instead of dread | Add "something slightly wrong" to every atmosphere line |
| Story is only 4,000 words | No word count targets set | Phase 3 sets targets: Act 1=2k, Act 2=4k, Act 3=2k |
| Twist feels random | No foreshadowing planned | Plant 2+ clues in Phase 3 architecture |
| Supporting chars are names only | Didn't plan their scenes | Phase 2 requires 2-3 scenes per character |

## WORKFLOW PRINCIPLES

### 1. Sequential Execution
Phases must be done in order. Phase 4 (writing) without Phase 3 (architecture) produces weak stories.

### 2. Verify Before Proceeding
Each phase has an output checklist. If any item fails, fix it before moving forward.

### 3. Templates Over Freestyle
Use provided templates. They capture 50+ stories of learnings and prevent common mistakes.

### 4. Read Then Write
Read the entire phase document before starting that phase's work.

### 5. Quality Over Speed
Taking 8 hours to produce an excellent story is better than 2 hours for a mediocre one.
