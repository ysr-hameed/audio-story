## ATMOSPHERE — Every 300-500 Words

**Atmosphere = sight detail + sound texture + physical feeling + protagonist's reaction.**

Atmosphere must CHANGE as the story progresses:

| Story Phase | Atmosphere Type | Example |
|-------------|----------------|---------|
| Act 1 (normal) | Calm, routine, familiar sounds | Fans humming, clock ticking, tea kettle |
| Act 2 (escalation) | Unease, wrongness, subtle changes | Flickering lights, silence where there should be sound |
| Act 3 (climax) | Threatening, oppressive, wrong | Distorted sounds, darkness, cold, pressure |

```
✅ ATMOSPHERE IN ACT 1 (Warm/familiar):
"रात का वक्त था। बारिश हो रही थी। ठंडी हवा के झोंके आ रहे थे। मैंने अपने कंधे पर जैकेट ली और दरवाजे की तरफ बढ़ा।"

✅ ATMOSPHERE IN ACT 2 (Creeping unease):
"उस रात बारिश थम गई थी। लेकिन हवा और भी ठंडी थी। लाइटें झपक रही थीं। मैंने देखा — बिजली तो नहीं जा रही। फिर ये लाइटें क्यों झपक रही हैं?"

✅ ATMOSPHERE IN ACT 3 (Full threat):
"अंधेरा था। बिजली चली गई थी। सिर्फ मेरी टॉर्च की रोशनी थी। वो रोशनी दीवार पर मेरी परछाई डाल रही थी। लेकिन परछाई हिल नहीं रही थी। मैं हिल रहा था। परछाई स्थिर थी।"
```

### ⚠️ ATMOSPHERE INTENSITY — DREAD, NOT DESCRIPTION

**The #1 atmosphere mistake: rich setting details but zero dread.**

Atmosphere is NOT:
- "Library thi. Kamre mein kitabein thi. Fan chal raha tha." (boring)
- "Raat ka waqt tha. Andhera tha." (generic)

Atmosphere IS:
- "Kitabon ki smell thi — sadi hui kagazon ki. Fan ki awaaz aati thi — lekin aaj woh awaaz alag lag rahi thi. Jaise koi aur bhi saans le raha ho." (dread)

```
❌ WEAK ATMOSPHERE (description only):
"Library mein andhera tha. Pankha chal raha tha. Kitabein racking par thin."

✅ STRONG ATMOSPHERE (dread):
"Library mein sirf pankhe ki awaaz thi — lekin aaj woh awaaz alag lag rahi thi.
Jaise koi aur bhi saans le raha ho. Main ne ghum kar dekha — koi nahi tha.
Lekin pankha band ho gaya. Ab sirf sannata tha."
```

**Rule: Every atmosphere line must make the listener feel UNEASE, not just SEE the setting.**
- Add something slightly WRONG
- Add a sound that shouldn't be there
- Add silence where there should be sound
- Add a physical reaction (goosebumps, cold sweat, tight chest)

### Atmosphere Between Events

**WARNING: Atmosphere commonly DROPS after Act 1.** The first scene is rich with sensory details, then the mystery takes over and the environment disappears. This flattens the story.

**Fix: After every major plot event, insert 1-2 atmosphere lines before moving to the next event.**

```
❌ NO ATMOSPHERE LINES BETWEEN EVENTS:
"मैंने शीशी उठाई। उसमें एक कागज था। मैंने पढ़ा। फिर मैं फाइल ढूंढने गया।"

✅ ATMOSPHERE BETWEEN EVENTS:
"मैंने शीशी उठाई। वो ठंडी थी — बहुत ठंडी। मैंने उसे खोला। अंदर एक कागज था।
फार्मेसी की लाइटें झपक रही थीं। वो आवाज — जो पंखों की होती है — वो भी थम गई थी।
सन्नाटा था। सिर्फ मेरी सांसों की आवाज। मैंने कागज पढ़ा।"
```

**Atmosphere must include at least ONE sensory detail every 300-500 words.**
Check: is there a SIGHT? SOUND? PHYSICAL FEELING? If any is missing, ADD IT.

## HINDI LANGUAGE RULES (MANDATORY)

### Pronoun Rules — Strict

| Correct | Incorrect | When to Use |
|---------|-----------|-------------|
| ये | याह | This person/object (singular) |
| वो | वाह | That person/object (singular) |
| ये | - | These people/objects (plural) |
| वे | वो (can use वो only in dialogue) | Those people/objects (plural) |

**Narration must use standard Hindi:**
- ❌ "वो किताब पुरानी थी।" (if you mean "that book")
- ✅ "वो किताब पुरानी थी।"
- ✅ "ये किताबें पुरानी हैं।" (ये for plural — correct)

**Dialogue may use colloquial forms** for natural speech, but narration must always follow standard grammar.

### Incorrect Phonetic Spellings — NEVER Use

| ❌ Wrong | ✅ Correct |
|----------|-----------|
| वाह (when वो is intended) | वो |
| याह (when ये is intended) | ये |
| उन्का | उनका |
| इस्में | इसमें |

### Grammar & Proofreading Checklist

Every paragraph must pass these checks before proceeding:
1. **Spelling:** No missing matras, no wrong halant usage, no chandrabindu where simple nasal is needed
2. **Grammar:** Subject-verb agreement, noun-adjective agreement (gender, number)
3. **Tense:** Consistent tense throughout (past tense for narration)
4. **Punctuation:** Full stops (।) at sentence ends, proper comma (,) pauses, em-dash (—) for self-talk
5. **Consistency:** Same spelling for same word throughout the story

## DETAILED NARRATION RULE — NO Word-by-Word Style

**Every action must be described with full sensory detail.** Never write short, disconnected clauses:

```
❌ BAD (word-by-word, no detail):
मैं वहां गया। दरवाजा खोला। अंदर देखा। कुछ मिला। डर गया।

❌ BAD (pipe-separated, fragmented):
मैं वहां गया | दरवाजा खोला | अंदर देखा | कुछ मिला | डर गया

✅ GOOD (detailed, flowing):
मैं उस पुराने दरवाजे के सामने खड़ा था। मेरे हाथ कांप रहे थे। मैंने धीरे-धीरे दरवाजे को धक्का दिया — वो चरमराता हुआ खुल गया। अंदर अंधेरा था। बहुत अंधेरा। मैंने टॉर्च ऑन की। रोशनी में धूल के कण तैर रहे थे। कमरे के बीच में एक पुराना बक्सा रखा था। मेरा दिल तेजी से धड़कने लगा। मैं जानता था — ये वही चीज है जिसे मैं ढूंढ रहा था।
```

**Each sentence must:**
- Build on the previous one (create a scene, not a list)
- Include protagonist's PHYSICAL REACTION (हाथ कांपना, दिल धड़कना, पसीना आना)
- Include ENVIRONMENTAL DETAIL (रोशनी, आवाज, गंध, तापमान)
- Move the action forward with complete descriptions

## WRITING STYLE (TTS-Friendly Hindi)

### Sentence Rules

- Short sentences (10-20 words). No long compound sentences.
- One idea per sentence. Each sentence moves forward.
- Use commas to create natural pausing.

```
❌ "मैंने डायरी खोली और उसमें एक पन्ना देखा जिस पर कुछ लिखा था।" (too compound)
✅ "मैंने डायरी खोली। एक पन्ना था। उस पर कुछ लिखा था।" (short, clear)
```

### Emotional Fillers — USE FREQUENTLY

```
Curiosity: "अरे!", "क्या बात है!", "ये क्या है?"
Surprise: "हे भगवान!", "वाह!", "कमाल की बात है!"
Fear: "मेरे होश उड़ गए", "रोंगटे खड़े हो गए"
Shock: "दिमाग घूम गया", "आंखें खुली की खुली रह गईं"
Physical: "हाथ कांपने लगे", "पैरों तले जमीन खिसक गई"
Frustration: "अरे यार!", "ये क्या हो रहा है?"
```

### Self-Talk — At Every Major Decision Point

```
✅ "मैं सोचने लगा — अरे ये कैसे हो सकता है? ये डायरी तो कल की बात लिखती है। ये तो मुमकिन नहीं।"
✅ "मैंने खुद से कहा — विजय, समझ ले। ये सब तेरे दिमाग का खेल है। वापस आ जा।"
```

### Dialogue Format

Supporting characters must DIALOGUE, not just be described:

```
❌ WEAK: "मोहन ने मुझसे पूछा कि मैं ठीक हूं या नहीं।"
✅ STRONG: मोहन बोला — "शंकर जी, आप ठीक हैं? आपकी आंखों में नींद नहीं है।"
```

## TTS OPTIMIZATION FOR PIPER MODEL

**Piper is a low-resource TTS model. It needs carefully formatted text for best results.**

### ⚠️ SCRIPT vs TONE — IMPORTANT DISTINCTION:

**SCRIPT:** देवनागरी (Devanagari) — Always write in Hindi script
**TONE:** Conversational/Hinglish — Write like people speak naturally

```
✅ RIGHT: देवनागरी script + conversational tone
"अरे ये क्या है? मैंने तो कुछ कहा नहीं। ये घोषणा कहां से आई?"

❌ WRONG: Roman script
"Are ye kya hai? Maine to kuch kaha nahi."

❌ WRONG: Formal literary Hindi (too stiff for TTS)
"अरे यह क्या है? मैंने तो कुछ कहा नहीं था। यह घोषणा कहां से आई?"
```

### CRITICAL RULES:

1. **Script = देवनागरी only** — Never use Roman script
2. **Tone = Conversational** — Write like people speak (use "ये" not "यह", use emotional fillers)
3. **ZERO English words allowed** — Everything must be in Hindi (even technical terms)
4. **Proper spacing** — Sentences should breathe (pause points for natural speech)
5. **Emotional markers** — Use fillers to guide tone
6. **No complex vocabulary** — Use simple everyday Hindi

### English to Hindi Replacements:

**Rule: NO English words in story text at all.**

**Strategy 1: Use commonly-borrowed Hindi-ized words (natural in conversation):**
```
✅ स्टेशन (station)
✅ प्लेटफार्म (platform)  
✅ ट्रेन (train)
✅ मोबाइल (mobile)
✅ माइक (microphone)
✅ केबिन (cabin)
✅ ड्यूटी (duty)
```

**Strategy 2: DESCRIBE instead of using English technical terms:**
```
❌ "मैंने recording system check किया"
✅ "मैंने देखा — क्या मेरी आवाज़ पहले से यंत्र में थी"
✅ "मैंने जांचा — कोई पुरानी आवाज़ तो नहीं बजी"

❌ "system में automatic playback था"  
✅ "यंत्र अपने आप आवाज़ बजा रहा था"

❌ "technical problem है"
✅ "कोई खराबी है"
✅ "कुछ गड़बड़ है"

❌ "speaker से आवाज़ आई"
✅ "ध्वनि-यंत्र से आवाज़ आई" OR "आवाज़ सुनाई दी"

❌ "I pressed play button"
✅ "मैंने बटन दबाया"

❌ "recording started"
✅ "आवाज़ का लेखन शुरू हुआ" OR "रिकॉर्डिंग शुरू हुई" (acceptable)
```

**Strategy 3: For unavoidable English words, use Hindi alternatives:**
```
❌ English          ✅ Hindi (Conversational)
──────────────────────────────────────────
recording           रिकॉर्डिंग (acceptable) OR "आवाज़ को सुरक्षित करना"
system              यंत्र / व्यवस्था
check               देखना / जांचना
problem             खराबी / समस्या
glitch              खराबी / गड़बड़
normal              ठीक / सामान्य
automatic           अपने आप
junior              जूनियर (acceptable) OR कनिष्ठ
senior              सीनियर (acceptable) OR वरिष्ठ
shift               शिफ्ट (acceptable) OR पाली
```

### TTS-Specific Formatting:

**1. Spacing for Natural Pauses:**
```
✅ GOOD (natural breathing):
मैंने देखा। वो वहां था। मैं हैरान रह गया।

❌ BAD (too fast, no pause):
मैंने देखा वो वहां था मैं हैरान रह गया।
```

**2. Sentence Length for TTS:**
- Maximum 15-20 words per sentence
- Use full stops (।) frequently
- Use commas (,) for natural pauses
- Use em-dash (—) for dramatic pause

**3. Emotional Pacing:**
```
✅ "अरे! ये क्या है?" (exclamation + pause + question)
✅ "मैं डर गया। बहुत डर गया।" (repetition for emphasis)
✅ "वो... वो वहां था।" (ellipsis for hesitation)
```

## TTS RULES — Numbers, Dates, Times

Piper TTS reads digit characters as individual digits. **All numbers MUST be spelled in Hindi:**

### 1. All numbers must be spelled in Hindi text:

```
2014 → दो हज़ार चौदह
1945 → एक हज़ार नौ सौ पैंतालीस
3:11 → तीन बजकर ग्यारह मिनट
2:50 → दो बजकर पचास मिनट
47-बी → सैंतालीस-बी
Room 4 → कमरा चार
12:47 → बारह बजकर सैंतालीस मिनट
```

### 2. All English words must be translated to Hindi:

**Common story words:**
```
night shift → रात की पड्यूटी / रात की शिफ्ट
duty → ड्यूटी / सेवा
cabin → केबिन / कमरा
clerk → लिपिक / क्लर्क
junior → जूनियर / कनिष्ठ
senior → सीनियर / वरिष्ठ
```

**Prefer commonly-used borrowed words over formal Sanskrit:**
- स्टेशन (not रेलवे-स्थानक)
- प्लेटफॉर्म (not चबूतरा)
- ट्रेन (not रेलगाड़ी - unless you want very formal tone)
- मोबाइल (not दूरभाष)

### 3. Final verification before TTS:

Search the story for:
- Any digit `[0-9]` → Convert to Hindi words
- Any Latin letter `[a-zA-Z]` → Translate or remove
- Any English word → Replace with Hindi equivalent

## TTS-FRIENDLY HINDI WORD SUBSTITUTIONS

Piper Hindi TTS struggles with certain aspirated sounds and formal constructions. Use these substitutions:

```
❌ Problem Word       ✅ TTS-Friendly         Why
─────────────────────────────────────────────────────
वाह!                  वा!                    'ह' ending sounds harsh/abrupt
हाँ                   हां                    chandrabindu causes mispronunciation
प्राप्त               मिला                   formal → conversational
किंतु                 लेकिन                  formal → conversational
अतः                   इसलिए                  formal → conversational
```

**Key substitutions:**
- Use "वा!" instead of "वाह!" for surprise/excitement
- Use "हां" instead of "हाँ" for agreement (no chandrabindu)
- In DIALOGUE only: natural speech can use "ये" and "वो" when characters speak casually
- In NARRATION always: "ये" (singular), "वो" (singular), "ये" (plural), "वे" (plural)
- Avoid chandrabindu (ाँ) wherever possible — use simple nasal (ां)
- Avoid visarga (ः) at word endings — rewrite the sentence

## WRITING PROCESS

### ⚠️ IMPORTANT: Write in Chunks for Long Stories

**AI models cannot write 7,000-10,000 words in one generation.**

Write the story in **3-4 separate chunks**:

**CHUNK 1: ACT 1 (~2,000 words)**
- Write complete Act 1
- Save to file
- Verify Act 1 checklist below

**CHUNK 2: ACT 2 Part 1 (~2,000 words)**
- Read existing file to maintain continuity
- Append to existing file
- Cover mechanism showings 1-3

**CHUNK 3: ACT 2 Part 2 (~2,000 words)**
- Read existing file
- Append to existing file  
- Cover mechanism showings 4-5 + moral dilemma

**CHUNK 4: ACT 3 (~2,000 words)**
- Read existing file
- Append to existing file
- Twist + confrontation + ending

### Write Act by Act:

**ACT 1 (~2,000 words):**
1. Write hook (first 300 words with mystery)
2. Weave backstory around hook (next 1,000 words)
3. First hint + testing scenes (500 words)
4. Check: All 6 backstory points covered?
5. Check: Atmosphere present with DREAD?
6. Check: Addiction Phase 1 (Curiosity) visible?

**ACT 2 (~4,000 words):**
1. Write mechanism showing #1-5 (800 words each)
2. Integrate supporting character scenes (2-3 per character)
3. Show addiction phases 2-7 explicitly
4. Write moral dilemma scene
5. Check: Real-world consequences after each event?
6. Check: Atmosphere evolving from Act 1?
7. Check: All supporting characters have dialogue?

**ACT 3 (~2,000 words):**
1. Write twist reveal
2. Write confrontation scene
3. Show addiction phase 8 (Panic)
4. Write haunting ending (2-3 lines)
5. Check: Twist was foreshadowed?
6. Check: Atmosphere at maximum threat level?
7. Check: Physical consequences shown?
